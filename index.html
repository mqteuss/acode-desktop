Configuração completa

Da instalação ao primeiro comando executado.

Siga a sequência uma única vez. O indicador acompanha a etapa que está na tela para você não se perder.

Etapa 1 de 10Baixar o ACode
Antes de começar
  • Windows 10 ou 11, 64 bits
  • ChatGPT web ou desktop com Plugins e Modo de desenvolvedor disponíveis
  • PowerShell ou Windows Terminal para instalar o cloudflared
  1. 1

    Baixe o ACode diretamente

    O botão abaixo aponta para o arquivo oficial da release mais recente. O download começa imediatamente, sem abrir a página do GitHub. Use o Setup para instalação normal; a versão Portable é opcional.

    Baixar para Windows ↓ Baixar versão Portable

    O navegador pode exibir o progresso na área de downloads. Não é necessário entrar no GitHub para instalar.

  2. 2

    Confira a integridade do arquivo

    Baixe o manifesto da mesma release e compare o SHA-256 no PowerShell.

    Get-FileHash .\ACode-Setup-x64.exe -Algorithm SHA256
    Baixar SHA256SUMS.txt
  3. 3

    Instale e abra o ACode

    Execute o Setup, escolha a pasta de instalação e conclua o assistente. Na primeira abertura, o servidor MCP fica restrito a 127.0.0.1.

  4. 4

    Instale o cloudflared pelo PowerShell

    Abra o PowerShell ou o Windows Terminal, cole o comando e aguarde a conclusão. O WinGet seleciona o pacote oficial pelo identificador exato.

    winget install --id Cloudflare.cloudflared -e --source winget

    Depois, feche e abra novamente o terminal e confirme a instalação:

    cloudflared --version
    Usar o instalador oficial manualmente

    O ACode não instala nem armazena credenciais da Cloudflare. Ele apenas inicia o executável instalado por você.

  5. 5

    Autorize somente a pasta do projeto

    No ACode, clique em Novo projeto, selecione a pasta exata do projeto e comece no modo Assistido. Não autorize o disco inteiro, a pasta do usuário ou diretórios de credenciais.

  6. 6

    Inicie o túnel HTTPS

    Abra Configurações → Conexão, selecione Cloudflare e inicie o túnel. Para testar, use o Quick Tunnel. O ACode mostrará uma URL HTTPS já contendo o caminho MCP protegido.

    Quick Tunnels recebem um endereço novo quando reiniciam. Para manter a mesma URL no ChatGPT, configure posteriormente um túnel nomeado e um hostname fixo.

  7. 7

    Copie o endpoint MCP completo

    Na tela de Conexão, use o botão de copiar. O formato será semelhante a:

    https://seu-tunel.example/mcp/SEU_TOKEN_ALEATORIO

    Trate essa URL como uma senha: não publique, não envie em prints e regenere o token se houver exposição.

  8. 8

    Ative o Modo de desenvolvedor no ChatGPT

    No ChatGPT, abra Configurações → Apps → Configurações avançadas e ative Modo de desenvolvedor. Em workspaces gerenciados, o administrador pode precisar liberar essa opção antes.

    Ver requisitos oficiais do Modo de desenvolvedor
  9. 9

    Abra Plugins e clique em Criar plugin

    No ChatGPT, entre na aba Plugins pela barra lateral ou por Configurações → Plugins. Clique em Criar plugin e use o fluxo para configurar a integração MCP do ACode.

    A OpenAI migrou o diretório de Apps para Plugins. Em algumas contas, a configuração técnica do servidor ainda aparece em Apps → Criar; use essa opção quando o ChatGPT encaminhar você para o app subjacente.

    Entender Plugins e Apps no ChatGPT

  10. 10

    Adicione o plugin ACode à conversa e faça o teste seguro

    Abra uma nova conversa, escolha o ACode no menu de Plugins ou mencione @ACode quando essa opção estiver disponível. Envie primeiro:

    Use o ACode para listar meus projetos autorizados. Não altere nenhum arquivo.

    Depois, peça para abrir um projeto e ler o contexto. Por último, crie um arquivo de teste. No modo Assistido, a alteração deverá aparecer no painel local para sua aprovação.

Segurança por arquitetura

O renderer não recebe acesso irrestrito ao computador.

As operações privilegiadas ficam no processo principal e passam por validação de schema, política de risco, permissão do projeto e auditoria.

Workspace confinado

Caminhos absolutos, traversal, hard links suspeitos e escapes por symlink são bloqueados.

Arquivos sensíveis

.env, chaves privadas e diretórios de credenciais são protegidos por padrão.

Endpoint protegido

Servidor local em loopback, token aleatório de 192 bits e comparação em tempo constante.

Atualização verificada

Somente release estável, hosts do GitHub permitidos, tamanho limitado e SHA-256 obrigatório.

Recomendação inicialUse Git, mantenha backups e deixe o primeiro projeto no modo Assistido até entender o fluxo de aprovações.

Sobre os 128 MB

É um teto de proteção, não um limite para o seu computador.

O ACode permite que você escolha os valores em Configurações → Execução. Os tetos evitam que arquivos hostis ou logs infinitos esgotem memória e contexto.

Arquivo aceito por operação
Configurável até 1 GB por arquivo.
Leitura enviada ao ChatGPT
Configurável até 128 MB por chamada; não limita o projeto no disco.
Histórico do terminal
Configurável até 128 MB para cada canal de cada processo.
RAM, CPU e build
Não são limitados por esses campos; seguem os recursos normais do sistema.

Atualizações futuras

O próprio ACode avisa quando uma release nova estiver disponível.

  1. Consulta a release estável mais recente no repositório configurado.
  2. Exibe uma notificação discreta com versão e notas.
  3. Baixa somente após sua confirmação.
  4. Compara tamanho e SHA-256 antes de liberar o instalador.
  5. Abre o Setup e encerra o app para concluir a atualização.
